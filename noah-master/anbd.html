ERD(Entity Relationship Diagram)

ERD

InSeoul 서비스의 ERD는 사용자를 중심으로 관광, 음식점, 부동산, 커뮤니티 도메인이 연결되는 구조로 설계되었습니다.
각 도메인은 독립적인 테이블을 가지되, 리뷰, 스크랩, 좋아요와 같은 공통 사용자 행위는 연관 테이블을 통해 관리되도록 구성했습니다.

Food & Food Review Domain

음식점 정보는 외부 공공데이터 API를 통해 수집되는 특성을 고려해, 기본 정보와 사용자 리뷰 데이터를 분리해 설계했습니다.
food 테이블에는 음식점 이름, 주소, 전화번호와 같이 변동이 적은 기본 정보를 저장하고,
food_review 테이블에는 별점, 리뷰 내용, 카테고리 등 사용자 행위 중심의 데이터를 관리했습니다.

System Sequence Diagram

시퀀스 다이어그램

관광지를 기준으로 주변 음식점 정보를 제공하기 위해 외부 API와 내부 DB를 결합한 데이터 처리 흐름을 시퀀스 다이어그램으로 정리했습니다.
사용자가 관광지를 선택하면 카카오 로컬 API를 통해 주변 음식점의 전화번호를 조회하고,
해당 전화번호를 기준으로 내부 음식점 DB와 매칭하여 저장된 음식점 기본 정보를 가져옵니다.
이후 매칭된 전화번호를 사용해 카카오 이미지 API를 호출함으로써 이미지는 실시간으로 조회하고,
DB에서 조회한 정보와 결합하여 사용자에게 하나의 화면으로 제공합니다.

IA(Information Architecture)

서비스 구조도

InSeoul 서비스는 메인 페이지를 중심으로 관광, 부동산, MBTI 커뮤니티, 회원 기능으로 구성된 서비스입니다.
관광 페이지에서는 관광지 목록 -> 관광지 상세 -> 주변 음식점 및 리뷰 확인으로 이어지는 구조를 가지며,
제가 담당한 음식점 정보 및 리뷰 기능은 관광 페이지 하위의 Tour Detail -> Food Detail & Review 흐름에 포함되어 있습니다.
IA를 통해 서비스 전체 구조를 먼저 정의한 후, 각 페이지 간 역할과 책임을 명확히 구분하여 기능별 구현과 화면 설계를 진행했습니다.

Flow Chart

기능 플로우 차트

사용자가 관광지 상세 페이지에서 음식점 카드를 선택하면 음식점 상세 정보 페이지가 표시됩니다.
이 과정에서 음식점 정보가 기존 DB에 저장되어 있는지 여부를 확인하고,
일치하는 정보가 존재하는 경우 리뷰 또는 스크랩 기능을 할 수 있도록 설계했습니다.
사용자의 선택에 따라 리뷰 작성 또는 스크랩이 수행되며, 해당 결과는 DB에 반영됩니다.
리뷰 또는 스크랩을 수행하지 않은 경우에는 별도의 DB 처리 없이 플로우가 종료됩니다.

Problem & Solution

문제 상황

관광지를 기준으로 주변 음식점 정보를 제공하는 기능을 구현하면서, 여러 외부 API를 동시에 연동해야 했습니다.
음식점의 기본 정보(이름, 주소, 전화번호)는 서울시 공공데이터 API를 통해 수집한 후 DB에 저장해 관리했고,
이미지나 위치 기반 정보는 외부 API를 통해 실시간으로 조회하는 방식이었습니다.
리뷰 기능은 페이지 이동 없이 모달 기반 UI로 동작했기 때문에,
음식점 카드 클릭 시 모달이 열리며 DB에 저장된 음식점·리뷰 정보와 외부 API에서 가져온 이미지 및 위치 정보가 여러 비동기 요청을 통해 동시에 갱신되어야 했습니다.
이 과정에서 비동기 요청의 응답 시점이 달라 화면 상태가 어긋나거나 일부 데이터가 정상적으로 표시되지 않는 문제가 발생했습니다.

해결 방법

음식점 정보 중 DB에서 관리되는 데이터와 외부 API를 통해 실시간으로 조회되는 데이터를 역할 기준으로 명확히 분리하여 비동기 흐름을 정리했습니다.
음식점 카드 클릭 후 모달이 열린 시점을 기준으로, DB에 저장된 음식점 기본 정보와 리뷰 데이터를 먼저 조회하고,
이후 외부 API를 통해 이미지 정보를 비동기로 호출하도록 API 호출 순서를 조정했습니다.
또한 각 Ajax 요청의 success 콜백 내부에서만 DOM이 변경되도록 UI 제어 방식을 통일해, 응답 시점 차이로 인한 모달 UI 상태 불일치 문제를 줄였습니다.
디버깅 도구를 활용해 DB API와 외부 API 요청 흐름을 구분해 추적하며, 비동기 처리 과정을 단계적으로 디버깅했습니다.
